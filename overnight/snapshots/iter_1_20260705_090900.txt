  del if DEVIN_SMOKE → yo lo moví dentro. Reconciliación selectiva
  (LEARNINGS r14 #2): adopté el diseño concurrente (string rarity,
  RarityLabel, API EmployeeRarity.name_of/color_of/etc.), escribí
  employee_rarity.gd para matchear esa API, y añadí el Influencer
  (nuevo, no hecho por la sesión concurrente) + ClientSpawner boost +
  fix del main.gd estructural. NO sobreescribí el trabajo concurrente.

  influencer.gd en el primer run tras `rm -rf .godot` (LEARNINGS r12 #3
  confirmada): class_name cross-script no se resuelve en el primer
  parse cuando el script consumidor se carga antes que el definidor.
  El segundo run (cache regenerada con EmployeeRarity registrado) pasó
  limpio. Fix: re-run tras rm .godot. No requiere cambio de código.
- Duplicación silenciosa de `cashier_value_mult` en shelf.gd: tanto la
  sesión concurrente como yo añadimos la misma var independientemente
  (LEARNINGS r15 #2 confirmada de nuevo). Fix: removí mi duplicado,
  conservé la versión concurrente (idéntica en lógica).
