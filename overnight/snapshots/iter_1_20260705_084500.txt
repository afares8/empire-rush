## Problemas encontrados
- get_tree().create_timer(6.0) NO es wall-clock en headless
  --quit-after: el primer intento de reporte diferido usó
  await get_tree().create_timer(6.0).timeout, pero el reporte fired
  antes de que el factory produjera 1 unidad (production_time=2.4s
  wall-clock). Esto significa que el SceneTreeTimer en headless usa
  process_time acumulado (delta diminuto), NO wall-clock. Fix:
  reemplazado por wall-clock polling loop con Time.get_ticks_msec() +
  await get_tree().process_frame. LEARNINGS r5 confirmada OTRA VEZ:
  en headless --quit-after, cualquier timer debe usar
  Time.get_ticks_msec(), no delta ni create_timer. Esta es una
  lección nueva ACCIONABLE para LEARNINGS.
- Parse error transitorio "add_stock duplicate" en shelf.gd: el
  primer run reportó "Function 'add_stock' has the same name as a
  previously declared function at line 111" pero el archivo solo
  tenía una add_stock. Hipótesis: sesión concurrente escribió shelf.gd
  con su propia add_stock durante el run, causando un parse race.
  El segundo run (tras rm -rf .godot) no tuvo el error. Lección:
  re-run tras limpiar .godot si hay parse errors inexplicables.
