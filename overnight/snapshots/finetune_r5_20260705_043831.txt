# Snapshot — Fine-tuning ronda 5
Fecha: 20260705_043831

## Lecciones extraídas

### Proceso
1. CRÍTICO: el controller hace timeout a 45 min y mata el proceso
   devin ANTES del `git commit` — las rondas 3, 4, 5 perdieron
   TODO su trabajo (LOOP-7/8/9 implementados 3 veces, 0 en git).
2. El controller no limpia el working tree entre rondas — el
   header "clean" del prompt era stale; rondas 3/4/5 encontraron
   WIP sin committar de la ronda anterior.
3. Solo 1 iteración por ronda (timeout 45 min muy corto para
   items M).
4. Anti-patrón "Devin huérfano concurrente" recurre a pesar del
   watchdog (r3 log: PID 4288 ~2h activo).
5. Snapshots faltantes: 4 de 5 esperados. Rondas 3/4/5 no
   escribieron snapshot (timeout las mató antes).

### Técnicas
1. `groups` override en .tscn no aplica a nodos instanciados en
   Godot 4.3 — usar NodePath a contenedor Node2D.
2. `theme_override_colors/font_color` no asignable en .tscn —
   usar `add_theme_color_override` en script.
3. MSYS PID ≠ Windows PID — usar `Get-Process` para taskkill.
4. Headless `--quit-after` no valida `_physics_process` ni
   `body_entered` — solo boot + `_process`.

### Diseño
1. El loop base está conectado en código pero NO se siente
   adictivo: sin HUD (LOOP-8), sin pad (LOOP-7), sin misión
   guiada (LOOP-9), sin export HTML5.
2. Sin export HTML5 no hay MVP jugable — todo smoke es headless
   y no valida feel real.
3. El gate "headless OK" es insuficiente — falta gate "export
   HTML5 OK + smoke en navegador OK".
